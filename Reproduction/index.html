1. 环境配置

1.1 关于 uv

uv 是一个极速的 Python 包管理器和项目管理工具，由 Rust 编写。它旨在替代 pip、pip-tools 和 virtualenv，提供比传统工具快 10-100 倍的速度。

也可以使用 conda

1.2 安装 uv

官方安装指南: https://docs.astral.sh/uv/getting-started/installation/
如果服务器无法直接连接外网或没有 root 权限，推荐直接使用 pip 安装： 

pip install uv

1.3 项目初始化步骤

配置好 uv 后，请按照以下步骤复现环境：

第一步：初始化项目与环境

初始化项目并创建虚拟环境（指定 Python 版本）。
注意：如果前一步已经手动 install 了 python（见下方 FAQ），这里会直接使用缓存。

uv init -p 3.12.4 gdkvm
uv venv --python 3.12.4
常见问题 (FAQ)：uv init 或 uv venv 报错

问题描述： 某些服务器（特别是内网环境）在尝试下载特定版本的 Python 解释器时，可能会因为网络限制或 SSL 证书过期导致下载失败（Failed to download python / invalid peer certificate）。

解决方案： 使用镜像源手动安装指定的 Python 版本（以 3.10.13 为例）：

# 1. 查看可用的 Python 版本
uv python list

# 2. 使用镜像源安装指定版本 (解决证书/网络问题)
uv python install 3.10.13 --mirror https://github-proxy.lixxing.top/https://github.com/astral-sh/python-build-standalone/releases/download

安装成功后，再次运行上面的 uv venv 命令即可。

第二步：进入项目目录
cd data/username/Repo/gdkvm
第三步：激活虚拟环境
source .venv/bin/activate
which python
第四步：克隆代码仓库
git clone https://github.com/wangrui2025/GDKVM.git
第五步：同步依赖
uv sync
uv pip check

2. 数据准备

我们主要使用 CAMUS 和 EchoNet-Dynamic 数据集。

2.1 数据集下载

已处理的数据
  • CAMUS: camus_png256x256_10f_20250709
  • EchoNet: EchoNet-Dynamic
CAMUS (原始数据)
  • 官方: CAMUS Challenge (Project Page)
  • Huggingface: CAMUS_public
  • Kaggle: 不推荐 camus-public-20250702
EchoNet-Dynamic (原始数据)
  • 官方下载: Stanford AIMI
  • Huggingface: -
  • Kaggle: 不推荐

2.2 数据组织结构

请下载数据集并按如下方式组织目录结构：

./dataset
├── CAMUS
└── EchoNet-Dynamic

3. 模型训练

要在 CAMUS 数据集上训练 GDKVM 模型，请运行以下命令。检查点（Checkpoints）将保存在 ./checkpoints 目录下。 

python train.py --dataset camus --batch_size 8 --epochs 100 --lr 1e-4 --save_dir ./checkpoints/camus_gdkvm

4. 推理与评估

要评估预训练模型，请运行： 

python test.py --dataset camus --resume ./checkpoints/camus_gdkvm/best_model.pth --eval_metrics DSC HD95
